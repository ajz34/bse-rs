# Basis Set Exchange in Rust (bse-rs) - Developer Guide

This is a Rust reimplementation of the Python [Basis Set Exchange](https://github.com/MolSSI-BSE/basis_set_exchange/) library. It provides APIs for retrieving, manipulating, and converting Gaussian-type orbital (GTO) basis sets for computational chemistry.

---

> **📋 IMPLEMENTATION PLAN**: See [PLAN.md](.claude/PLAN.md) for a detailed roadmap of missing features and their implementation status. This plan was created by comparing the Rust implementation against the Python BSE library.

---

## Environment Setup

### Required Environment Variable

You need to make sure your python environment contains `basis-set-exchange` package (can be installed via pypi).

If you do not have the package installed, you can set the `BSE_DATA_DIR` environment variable to point to the data directory of the Python BSE repository:

```bash
# Adjust this path to match the location of the basis_set_exchange repository on your system
export BSE_DATA_DIR=/home/a/Git-Others/basis_set_exchange/basis_set_exchange/data
```

This points to the Python BSE data directory containing basis set JSON files.

### Key Data Structures

**`BseBasis`** - Complete basis set information:
- `elements`: HashMap of atomic numbers → `BseBasisElement`
- `function_types`: List of function types (gto, gto_spherical, scalar_ecp)
- `name`, `description`, `family`, `role`, `version`

**`BseBasisElement`** - Per-element basis data:
- `electron_shells`: List of `BseElectronShell`
- `ecp_potentials`: ECP data (optional)
- `references`: Reference information

**`BseElectronShell`** - Individual shell:
- `angular_momentum`: AM values (e.g., `[0]` for s, `[0,1]` for sp)
- `exponents`: Gaussian exponents
- `coefficients`: Contraction coefficients

### Main API Functions

```rust
use bse::prelude::*;

// Get basis set as JSON-like structure
let args = BseGetBasisArgsBuilder::default()
    .elements("1, 6-O")  // H, C through O
    .build()
    .unwrap();
let basis: BseBasis = get_basis("cc-pVTZ", args);

// Get formatted output for specific software
let args = BseGetBasisArgsBuilder::default()
    .elements("H, C-O")
    .header(false)
    .build()
    .unwrap();
let output: String = get_formatted_basis("cc-pVTZ", "nwchem", args);

// Read basis from formatted string
let basis_minimal: BseBasisMinimal = read_formatted_basis_str(&output, "nwchem");
```

### BseGetBasisArgs Options

| Option | Type | Description |
|--------|------|-------------|
| `elements` | `Option<String>` | Element selection (atomic numbers, symbols, ranges) |
| `version` | `Option<String>` | Specific version of basis set |
| `uncontract_general` | `bool` | Remove general contractions |
| `uncontract_spdf` | `bool` | Split sp, spd, spdf combined shells |
| `uncontract_segmented` | `bool` | Fully uncontract (each primitive separate) |
| `make_general` | `bool` | Make as generally-contracted as possible |
| `optimize_general` | `bool` | Optimize general contractions |
| `remove_free_primitives` | `bool` | Remove uncontracted primitives |
| `augment_diffuse` | `i32` | Add n diffuse functions (even-tempered) |
| `augment_steep` | `i32` | Add n steep functions |
| `get_aux` | `i32` | 0: orbital, 1: AutoAux, 2: AutoABS |
| `header` | `bool` | Include header in formatted output |

## Development Workflow

### Adding a New Writer

1. Create new file `src/writers/<format>.rs`
2. Implement `write_<format>(basis: &BseBasis) -> String`
3. Add module in `src/writers/mod.rs`
4. Register in `src/writers/write.rs` `writer_map()` function
5. Add test case in writer file
6. Add integration test in `tests/test_get_basis.rs`
7. Generate Python reference file if needed

### Adding a New Reader

1. Create new file `src/readers/<format>.rs`
2. Implement `read_<format>(basis_str: &str) -> Result<BseBasisMinimal, BseError>`
3. Add module in `src/readers/mod.rs`
4. Register in `src/readers/read.rs` `reader_map()` function
5. Add test case and Python reference

### Adding New Manipulation

Manipulations are in `src/manip.rs`. Key functions:
- `prune_shell` / `prune_basis` - Remove duplicates and zero coefficients
- `uncontract_general` / `uncontract_segmented` / `uncontract_spdf`
- `make_general` / `optimize_general`
- `geometric_augmentation` - Add diffuse/steep functions
- `autoaux_basis` / `autoabs_basis` - Generate auxiliary basis sets

## Testing Strategy

### Reference-Based Testing

Tests compare Rust output against Python BSE v0.11 reference files:
- JSON comparisons use `assert_json_include!` (partial matching)
- Text comparisons use exact string equality
- Text diff visualization via `similar` crate

### Generating New Reference Files

If Python BSE is available:
```bash
cd tests/python_ref
python get_basis_json.py   # Generate JSON references
python get_basis_fmt.py    # Generate format output references
python read_basis_fmt.py   # Generate read test references
```

## Notes for Developers

1. **Data not bundled**: This crate only provides code; basis set data comes from Python BSE
2. **Direct translation**: Code closely mirrors Python structure for maintainability
3. **Caching**: File reads are cached via `#[cached]` macro for performance
4. **Error handling**: Uses `BseError` enum with `bse_raise!` macro
5. **Case insensitive**: Basis names and format names are case insensitive
